## 第一步：配置 Cloudinary（图片存储）

项目已集成 `cloudinary` SDK，只需获取一条 URL。

1. 登录 [Cloudinary 控制台](https://console.cloudinary.com)。
2. 在首页 **Dashboard** 找到 **API Environment variable**，格式为：
   ```
   CLOUDINARY_URL=cloudinary://API_KEY:API_SECRET@CLOUD_NAME
   ```
3. 复制 `cloudinary://...` 开头的完整字符串（去掉前面的 `CLOUDINARY_URL=`），记录备用。

> 若看不到完整密钥，点击页面上的眼睛图标展开。

---

## 第二步：在 Railway 部署后端 + 数据库

Railway 将后端服务和 PostgreSQL 数据库放在同一个项目里，数据库连接字符串会**自动注入**，无需手动配置。

### 2.1 创建 Railway 项目并添加 PostgreSQL

1. 登录 [Railway Dashboard](https://railway.app/dashboard)。
2. 点击 **New Project** → **Deploy from GitHub repo**，选择你的仓库。
3. Railway 检测到 Python 项目后会尝试自动部署，先**不要着急**，继续下一步先添加数据库。
4. 在项目页面点击右上角 **+ New** → **Database** → **Add PostgreSQL**。
5. Railway 会在同一项目内创建一个 PostgreSQL 实例，并自动将 `DATABASE_URL` 环境变量注入到后端服务，无需手动填写。

### 2.2 配置后端服务

点击 Railway 项目中的后端服务（非数据库），进入服务设置：

**① 设置根目录**

进入服务 → **Settings → Source** → 将 **Root Directory** 设为 `backend`。

**② 设置启动命令**

进入服务 → **Settings → Deploy** → 将 **Start Command** 设为：

```
uvicorn main:app --host 0.0.0.0 --port $PORT
```

> Railway 通过 `$PORT` 注入端口，不要硬编码 `8001`。

**③ 添加环境变量**

进入服务 → **Variables** 标签页，点击 **+ New Variable** 逐一添加：

| 变量名 | 值 |
|--------|----|
| `SECRET_KEY` | 准备工作中生成的随机字符串 |
| `ACCESS_TOKEN_EXPIRE_MINUTES` | `60` |
| `ALLOWED_ORIGINS` | `http://localhost:5173`（稍后前端上线后补充） |
| `CLOUDINARY_URL` | 第一步中复制的完整 URL，如 `cloudinary://key:secret@cloud_name` |
| `DEEPSEEK_API_KEY` | 你的 DeepSeek API Key（可选，不使用 AI 功能则留空） |

> `DATABASE_URL` **无需手动添加**，Railway 已自动注入。可在 Variables 页面确认是否存在该变量。

**④ 触发部署**

所有配置填完后，进入 **Deployments** 标签页，点击 **Deploy** 或等待 Railway 自动触发构建。

### 2.3 记录后端 URL

部署成功后，进入服务 → **Settings → Networking** → 开启 **Public Networking**，Railway 会分配一个形如：

```
https://tongai-backend.up.railway.app
```

的公网 URL，记录备用。

访问 `https://tongai-backend.up.railway.app/health`，返回 `{"status":"ok"}` 说明部署成功。

## 第三步：执行数据库迁移

后端部署成功后，需要在线上创建数据库表结构。

1. 在 Railway 后端服务页面，点击顶部 **Shell** 标签页（若无此选项，需先在 Settings 中开启）。
2. 在 Shell 中依次执行：

   ```bash
   # 执行迁移，创建所有数据表
   alembic upgrade head
   ```

   看到如下输出表示成功：
   ```
   INFO  [alembic.runtime.migration] Running upgrade  -> 730c04db0bb9, init all tables
   ```

3. 创建后台管理员账号：

   ```bash
   python create_admin.py
   ```

   按提示输入用户名和密码，记录好备用。

## 第四步：部署前端到 Vercel

### 4.1 在 Vercel 创建项目

1. 登录 [Vercel Dashboard](https://vercel.com/dashboard)。
2. 点击 **Add New Project**，导入 GitHub 仓库。
3. 填写配置：

   | 字段 | 值 |
   |------|----|
   | **Framework Preset** | `Vite`（Vercel 会自动检测） |
   | **Root Directory** | `frontend` |
   | **Build Command** | `npm run build` |
   | **Output Directory** | `dist` |
   | **Install Command** | `npm install` |

4. 在 **Environment Variables** 中添加：

   | 变量名 | 值 |
   |--------|----|
   | `VITE_API_BASE_URL` | `https://tongai-backend.up.railway.app/api/v1` |

   （将 URL 替换为第二步中记录的你的实际后端地址）

5. 点击 **Deploy**，等待构建完成。

### 4.2 记录前端 URL

部署成功后，Vercel 会分配形如 `https://tongai.vercel.app` 的域名，记录备用。

## 第五步：回填 CORS 配置

前端上线后，需要让后端允许该域名的跨域请求。

1. 回到 **Railway → 后端服务 → Variables** 标签页。
2. 找到 `ALLOWED_ORIGINS` 变量，点击编辑，将值改为：
   ```
   http://localhost:5173,https://tongai.vercel.app
   ```
   （用英文逗号分隔，**不要有空格**，将 Vercel 域名替换为你的实际域名）
3. 保存后 Railway 会自动重新部署后端。

## 第六步：绑定自定义域名（可选）

如果你有自己的域名（如从阿里云购买），可以免费绑定：

### 前端（Vercel）

1. Vercel 项目 → **Settings → Domains** → 输入你的域名并添加。
2. 按提示在 DNS 服务商处添加 `CNAME` 记录，指向 `cname.vercel-dns.com`。
3. Vercel 自动颁发 HTTPS 证书，生效约需 1～5 分钟。

### 后端（Railway）

1. Railway 服务 → **Settings → Networking → Custom Domain** → 输入域名。
2. 按提示在 DNS 服务商处添加对应记录。
3. Railway 自动颁发 HTTPS 证书。

绑定完成后，同步更新以下两处：
- Railway `ALLOWED_ORIGINS` → 加入前端自定义域名
- Vercel `VITE_API_BASE_URL` → 改为后端自定义域名，并触发重新部署

## 日常更新流程

```bash
# 修改代码后，推送到 GitHub
git add .
git commit -m "feat: ..."
git push origin main

# Vercel 和 Railway 均配置了 GitHub 自动部署，推送后自动构建，无需手动操作
```

**如果修改了数据库模型（`models.py`）**，还需要额外操作：

1. 本地生成迁移脚本：
   ```bash
   cd backend
   alembic revision --autogenerate -m "描述本次变更"
   ```
2. 将生成的迁移文件一起提交推送。
3. 推送后，在 **Railway Shell** 中执行：
   ```bash
   alembic upgrade head
   ```

## 常见问题排查

### Railway 部署失败，提示找不到 `main.py`

检查服务的 **Root Directory** 是否设置为 `backend`。Railway 默认从仓库根目录查找，必须指定子目录。

### 前端请求 API 返回 CORS 错误

检查 Railway 中 `ALLOWED_ORIGINS` 变量是否包含当前前端的完整域名（含 `https://`），且没有多余空格或末尾斜杠。

### API 请求返回 500，日志显示数据库连接失败

确认 Railway 项目中 PostgreSQL 服务与后端服务在**同一个 Project** 内，且后端服务的 Variables 中存在自动注入的 `DATABASE_URL`。

### Alembic 迁移报错 `Target database is not up to date`

说明本地有未同步的迁移版本，先在本地执行 `alembic upgrade head`，再生成新的迁移文件。

### 图片上传失败

在 Railway Variables 中确认 `CLOUDINARY_URL` 已正确填写（格式为 `cloudinary://key:secret@cloud_name`），无多余空格。可在 Railway Shell 中执行以下命令验证：
```bash
python -c "import os; print(os.getenv('CLOUDINARY_URL'))"
```

### Railway 额度快用完了怎么办

在 Railway 控制台 → **Account Settings → Usage** 查看当月消耗。若接近 $5 上限，可以：
- 升级到 Railway Pro 计划按实际用量付费
- 在月底前将服务暂停（**Sleep**），次月重置后再恢复
