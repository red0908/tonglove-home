# 多智能体客户角色识别 —— 官网交互 Demo 方案三

> 文档版本：v1.0 | 创建日期：2026-05-06  
> 定位：将专利《一种基于多智能体决策的客户角色识别方法及系统》的核心技术，以最直观的交互方式在个人官网上呈现，作为技术实力展示的核心板块之一。

## 一、方案概述

### 1.1 目标

| 目标维度 | 说明 |
|---------|------|
| **技术展示** | 让访客亲身体验"多智能体协同决策"流程，而非仅阅读文字说明 |
| **可信度背书** | 附带专利号/技术要点，彰显原创性与深度 |
| **转化引导** | 体验后引导访客联系合作 / 进一步了解完整产品 |

### 1.2 核心体验路径

```
访客输入一段对话文本（或选择预设样本）
        ↓
前端实时展示「多智能体分析流程」动画（路由→分析→仲裁）
        ↓
输出：角色分类标签 + 置信度 + 证据链高亮
        ↓
可选：用户反馈纠正，触发"增量优化"演示
```

## 三、预设样本设计

### 样本 A：技术型客户（预期输出：架构师/技术决策者）

```
张工：你们这套中台的消息队列用的是什么方案，Kafka 还是自研？
      吞吐量峰值能到多少，延迟 SLA 怎么保证？

销售：我们底层用 Kafka，峰值 TPS 可以达到百万级，P99 延迟<10ms。

张工：那跨数据中心的灾备方案呢？我们有两地三中心的要求，
      目前我们自己做了 Mirror Maker 2，但同步延迟还是个问题。
```

> 预期结果：一级=技术导向，二级=架构师，置信度≈96%  
> 证据链：Kafka / TPS / P99 / Mirror Maker 2 / 两地三中心

### 样本 B：业务型客户（预期输出：采购/业务决策者）

```
李总：我们现在手动处理报表要花团队两天时间，你们能帮我们缩短到多久？

销售：接入我们的自动化模块后，通常可以缩短到 2 小时以内。

李总：ROI 怎么算？我需要一个能给 CFO 看的方案，
      另外合同签署周期你们一般是多久？
```

> 预期结果：一级=业务导向，二级=业务决策者，置信度≈91%  
> 证据链：ROI / CFO / 合同签署 / 手动处理

### 样本 C：混合型客户（角色动态转换场景）

```
王总：我们技术团队上周做了一轮测试，压测结果不太理想。（技术顾问视角）

销售：具体是哪个模块的问题，我们可以针对性优化。

王总：算了，技术问题让我们架构师跟进吧。
      我更关心的是，你们能不能在 Q3 前完成整体交付，
      这关系到我们年底的预算核销节点。（决策者视角）
```

> 预期结果：检测到角色转换（技术顾问→决策者），时序图谱高亮转折点

### 4.2 后端 API 设计

**接口：`POST /api/ai/role-identify`**

```typescript
// 请求体
interface RoleIdentifyRequest {
  text: string;          // 对话文本
  mode?: 'fast' | 'deep' | 'auto';  // 默认 auto（由路由智能体决定）
}

// 响应体（SSE 流式，逐步推送分析步骤）
interface RoleIdentifyStream {
  step: 'routing' | 'fast_analysis' | 'deep_research' | 'arbitration' | 'result';
  agent: string;         // 当前执行的智能体名称
  elapsed_ms: number;    // 当前步骤耗时
  data: {
    // step='routing' 时
    complexity_score?: number;
    route_decision?: 'fast' | 'deep';
    // step='result' 时
    primary_label?: string;       // 一级分类
    secondary_label?: string;     // 二级职能
    confidence?: number;          // 置信度 0-1
    evidence_chain?: EvidenceItem[];
    role_transition?: TransitionPoint[];  // 角色转换点
  };
}

interface EvidenceItem {
  text: string;          // 证据原文片段
  source: 'keyword' | 'rag' | 'context';
  weight: number;
}
```

## 五、交互动效细节

### 5.1 智能体节点状态机

```
idle → pending（闪烁蓝色边框）
     → processing（旋转 loading 图标 + 计时器）
     → completed（绿色 ✓ + 耗时 badge）
     → skipped（灰色，路由未选择此分支时）
```

### 5.2 证据链高亮动效

- 结果返回后，对话文本区域中的关键词依次渐入高亮（错落 100ms 间隔）
- 高亮颜色区分来源：
  - 黄色：关键词库匹配
  - 蓝色：RAG 语义检索
  - 橙色：上下文推断

### 5.3 角色转换时序轴（样本 C 专属）

在对话文本右侧展示迷你时序轴，标注角色从「技术顾问」转变为「决策者」的精确句子位置。
