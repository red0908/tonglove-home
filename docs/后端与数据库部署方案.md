## 二、本地 Docker PostgreSQL 配置

### 2.1 创建 Docker 容器

打开终端，执行以下命令启动一个本地 PostgreSQL 容器：

```bash
docker run --name personal-postgres \
  -e POSTGRES_USER=myuser \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  -v personal_postgres_data:/var/lib/postgresql/data \
  -d postgres:15
```

**参数解释**：
- `--name`：容器名称，便于管理。
- `-e`：设置环境变量（用户名、密码、默认数据库名）。
- `-p 5432:5432`：将宿主机的 5432 端口映射到容器的 5432 端口。
- `-v`：数据卷挂载，保证停止容器后数据不丢失。
- `-d`：后台运行。

### 2.2 检查容器状态

```bash
docker ps
```

应该看到 `personal-postgres` 容器状态为 `Up`。

### 2.3 连接测试（可选）

使用命令行工具 `psql` 连接（需要先安装 `psql` 客户端）：

```bash
docker exec -it personal-postgres psql -U myuser -d mydb
```

输入密码 `mypassword` 后出现 `mydb=#` 提示符表示成功。输入 `\q` 退出。

## 三、项目配置（FastAPI + SQLAlchemy）

### 3.1 安装依赖

在项目根目录（后端文件夹）创建虚拟环境并安装：

```bash
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

pip install fastapi uvicorn sqlalchemy alembic psycopg2-binary python-dotenv
```

安装完成后，导出 `requirements.txt`（Railway 部署依赖此文件）：

```bash
pip freeze > requirements.txt
```

> 也可以使用 `pip install pipreqs && pipreqs .` 只导出项目实际用到的依赖，文件更干净。

### 3.2 环境变量文件 `.env`

在项目根目录创建 `.env` 文件（这个文件不要提交到 Git，需加入 `.gitignore`）：

```env
# 本地开发使用 Docker PostgreSQL
DATABASE_URL=postgresql://myuser:mypassword@localhost:5432/mydb

# 可选：AI API Key
DEEPSEEK_API_KEY=your_key_here

# CORS 允许的前端域名（多个用逗号分隔）
ALLOWED_ORIGINS=http://localhost:5173
```

**注意**：线上 Railway 会自动注入 `DATABASE_URL` 环境变量，所以无需在代码中硬编码。

### 3.3 数据库配置文件 `database.py`

```python
from sqlalchemy import create_engine
from sqlalchemy.orm import DeclarativeBase, sessionmaker
import os
from dotenv import load_dotenv

load_dotenv()

SQLALCHEMY_DATABASE_URL = os.getenv("DATABASE_URL")
if not SQLALCHEMY_DATABASE_URL:
    raise ValueError("DATABASE_URL 环境变量未设置，请检查 .env 文件")

engine = create_engine(
    SQLALCHEMY_DATABASE_URL,
    pool_size=5,        # 连接池大小
    max_overflow=10,
    pool_pre_ping=True, # 断线自动重连
)

SessionLocal = sessionmaker(autocommit=False, autoflush=False, bind=engine)

class Base(DeclarativeBase):
    pass
```

### 3.4 定义模型（示例：用户表）

创建 `models.py`：

```python
from sqlalchemy import Column, Integer, String, Boolean, DateTime, func
from database import Base  # Base 在 database.py 中通过 DeclarativeBase 定义

class User(Base):
    __tablename__ = "users"

    id = Column(Integer, primary_key=True, index=True)
    username = Column(String(50), unique=True, nullable=False)
    hashed_password = Column(String(255), nullable=False)
    is_active = Column(Boolean, default=True)
    created_at = Column(DateTime(timezone=True), server_default=func.now())
```

### 3.5 测试数据库连接

创建 `test_db.py` 临时脚本：

```python
from database import SessionLocal, engine
from models import Base

# 创建表
Base.metadata.create_all(bind=engine)

# 插入测试数据
db = SessionLocal()
try:
    from models import User
    test_user = User(username="test", hashed_password="dummy")
    db.add(test_user)
    db.commit()
    print("数据库连接成功，已插入测试用户")
except Exception as e:
    print("错误:", e)
    db.rollback()
finally:
    db.close()
```

运行 `python test_db.py`，无报错说明 Docker PostgreSQL 连接正常。

## 四、使用 Alembic 管理数据库迁移

### 4.1 初始化 Alembic

```bash
alembic init migrations
```

### 4.2 修改 `alembic.ini`

找到 `sqlalchemy.url` 行，**保持默认占位符或填写任意值即可**，实际 URL 将由 `env.py` 从环境变量中读取（见 4.3），无需在此处硬编码真实密码：

```ini
sqlalchemy.url = driver://user:pass@localhost/dbname
```

> **安全提示**：`alembic.ini` 本身不要写入真实凭证。务必将其加入 `.gitignore`（或保持占位符不含真实密码后再提交），避免密码泄露。

### 4.3 修改 `migrations/env.py`

在 `env.py` 中找到 `target_metadata` 部分，改为动态读取数据库 URL：

```python
# 在文件开头添加
from dotenv import load_dotenv
import os
load_dotenv()

# 然后找到 run_migrations_offline 和 run_migrations_online 中的配置
# 将 sqlalchemy.url 改为从环境变量读取
config.set_main_option('sqlalchemy.url', os.getenv('DATABASE_URL'))

# 同时设置 target_metadata
from models import Base
target_metadata = Base.metadata
```

### 4.4 生成迁移脚本

```bash
alembic revision --autogenerate -m "init users table"
```

检查 `migrations/versions` 下生成的脚本是否正确。

### 4.5 执行迁移

```bash
alembic upgrade head
```

此时再去 PostgreSQL 中查看，应该已有 `users` 表。

## 五、线上部署（Railway PostgreSQL）

### 5.1 推送到 GitHub

确保 `.env` 已加入 `.gitignore`，然后推送代码：

```bash
git add .
git commit -m "feat: init backend with docker postgres"
git push origin main
```

### 5.2 在 Railway 上创建项目

1. 登录 [Railway](https://railway.app/)。
2. 点击 **New Project** → **Deploy from GitHub repo**。
3. 选择你的后端仓库。
4. Railway 会自动检测 Python 并尝试启动。如果启动失败（因为缺少数据库），先添加数据库。

### 5.3 添加 PostgreSQL 数据库

在项目页面点击 **New** → **Database** → **Add PostgreSQL**。Railway 会自动配置一个 PostgreSQL 实例，并自动注入环境变量 `DATABASE_URL` 到你的后端服务。

> **Railway 免费额度说明**：Hobby 计划每月约 $5 免费额度，PostgreSQL 存储上限约 1GB。对个人作品官网完全够用，但建议在 Railway 控制台开启用量告警，避免意外超额扣费。

### 5.4 设置其他环境变量

在 Railway 的 **Variables** 中添加：
- `DEEPSEEK_API_KEY`（如果用到 AI）
- 其他自定义变量

### 5.5 配置启动命令

Railway 通常会自动识别 `main.py` 中的 FastAPI app 并使用 `uvicorn` 启动。如果不生效，可以在项目根目录创建 `Procfile`：

```procfile
web: uvicorn main:app --host 0.0.0.0 --port $PORT
```

或者通过 Railway 的 **Settings** → **Start Command** 手动填写。

### 5.5.1 添加健康检查接口

Railway 部署时会通过 HTTP 探测服务是否正常运行，建议在 `main.py` 中加入：

```python
@app.get("/health")
def health_check():
    return {"status": "ok"}
```

并在 Railway 的 **Settings** → **Health Check Path** 中填入 `/health`。

### 5.6 执行数据库迁移（线上）

Railway 不会自动运行 `alembic upgrade head`，你需要通过以下方式之一执行：

#### 方法一：使用 Railway 的 Shell 功能（推荐）
1. 在 Railway 项目页面，点击你的后端服务 → **Shell** 选项卡。
2. 在打开的终端中运行：
   ```bash
   alembic upgrade head
   ```

#### 方法二：在代码启动时自动迁移（不推荐生产环境，因为多实例会冲突）
在 `main.py` 中添加：
```python
from alembic.config import Config
from alembic import command

alembic_cfg = Config("alembic.ini")
command.upgrade(alembic_cfg, "head")
```

#### 方法三：使用 Railway 的 Deploy Hook 脚本（高级）

### 5.7 测试线上接口

获取 Railway 分配给后端的公网 URL（形如 `https://your-backend.up.railway.app`），访问 `/docs` 查看 API 文档。如果能正常打开，说明数据库连接成功。

## 六、前端连接后端

### 6.1 前端环境变量

在 React 项目的 `.env.local` 中：

```env
VITE_API_BASE_URL=https://your-backend.up.railway.app/api/v1
```

### 6.2 跨域配置

在后端 `main.py` 中添加 CORS 中间件，允许前端 Vercel 域名和本地开发域名：

```python
from fastapi.middleware.cors import CORSMiddleware
import os

# 通过环境变量配置允许的域名，多个用逗号分隔
# 示例：ALLOWED_ORIGINS=http://localhost:5173,https://your-frontend.vercel.app
allowed_origins = os.getenv("ALLOWED_ORIGINS", "http://localhost:5173").split(",")

app.add_middleware(
    CORSMiddleware,
    allow_origins=allowed_origins,
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)
```

在 `.env` 中添加：

```env
ALLOWED_ORIGINS=http://localhost:5173,https://your-frontend.vercel.app
```

在 Railway Variables 中添加同名变量即可覆盖线上配置，无需修改代码。

## 七、日常开发与部署流程总结

| 操作 | 本地环境 | 线上环境 |
|------|----------|----------|
| **启动数据库** | `docker start personal-postgres` | Railway 自动运行 |
| **运行后端** | `uvicorn main:app --reload` | 推送代码自动部署 |
| **执行迁移** | `alembic upgrade head` | Railway Shell 中执行 `alembic upgrade head` |
| **查看数据** | `docker exec -it personal-postgres psql ...` | Railway 数据库页面或 `psql` 命令行 |
| **环境变量** | 本地 `.env` | Railway Variables |

## 八、常见问题与解决

1. **本地容器启动后，连接被拒绝**  
   - 检查 Docker 是否运行，端口是否被占用（`lsof -i :5432`）
   - 确认用户名密码与 `.env` 一致

2. **Alembic 迁移生成时提示“No changes detected”**  
   - 确认 `env.py` 中 `target_metadata` 正确导入了 `Base.metadata`
   - 确认已定义模型且正确继承 `Base`

3. **Railway 上执行 `alembic upgrade head` 报错 “No module named …”**  
   - 进入 Shell 后，先运行 `pip list` 确认依赖已安装。若缺少，运行 `pip install -r requirements.txt`（需要预先导出依赖）

4. **线上环境连接数据库失败**  
   - 检查 Railway 是否已经为后端服务绑定了 PostgreSQL（需要两者在同一个项目内）
   - 检查 `DATABASE_URL` 环境变量是否自动注入（在 Variables 中查看）

## 九、附：一键启动本地 Docker 的脚本

将以下内容保存为 `start_db.sh`（Linux/macOS）或 `start_db.bat`（Windows），方便每次启动：

**start_db.sh**:
```bash
#!/bin/bash
docker start personal-postgres || docker run --name personal-postgres \
  -e POSTGRES_USER=myuser \
  -e POSTGRES_PASSWORD=mypassword \
  -e POSTGRES_DB=mydb \
  -p 5432:5432 \
  -v personal_postgres_data:/var/lib/postgresql/data \
  -d postgres:15
echo "PostgreSQL started on port 5432"
```

赋予执行权限 `chmod +x start_db.sh`，每次运行即可。
