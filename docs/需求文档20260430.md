# 个人作品官网 - 需求设计文档（前后端分离）

## 1. 项目概述

### 1.1 项目定位
个人全栈AI开发者作品集官网，展示技术能力、项目经验、技术文章，并集成AI交互功能作为亮点。面向技术面试官、招聘方、技术同行的在线名片。

### 1.2 核心目标
- 展示个人品牌：8年全栈 + 2年AI落地经验、3项AI专利、企业创新奖
- 作品集模块：呈现AI项目、专利、技术成果
- 博文模块：分享技术思考，体现持续学习与表达能力
- AI体验扩展：提供可交互的AI Demo（RAG问答、Agent模拟等），证明工程落地能力
- 响应式设计，支持PC/移动端访问

### 1.3 非功能性需求
- 性能：首屏加载 ≤ 2秒（含API调用）
- 可用性：核心功能具备友好的错误提示
- 可维护性：前后端分离，代码结构清晰，支持后续扩展
- 安全性：管理后台需身份验证（JWT / 简单密码）；API防止恶意调用（限流可选）

## 2. 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | React 18 + Vite | 快速开发，支持热更新 |
| 样式 | TailwindCSS | 实用优先，响应式快速构建 |
| 路由 | React Router v6 | 客户端路由 |
| 状态管理 | Zustand / Context API | 轻量级，用于主题、用户登录态等 |
| HTTP请求 | Axios | 统一错误处理，拦截器配置 |
| 后端 | FastAPI (Python 3.10+) | 异步支持，自动API文档（/docs） |
| 数据库 | PostgreSQL（生产）+ SQLite（开发） | 关系型数据库，存储用户、作品、博文等 |
| ORM | SQLAlchemy 2.0 | 异步支持，模型定义清晰 |
| 身份验证 | JWT（`python-jose`）+ `passlib`（bcrypt） | 用于管理后台登录 |
| AI集成 | DeepSeek API / OpenAI API | 可配置，实现RAG问答、Agent演示 |
| 向量检索 | Chroma（可选） | 用于RAG Demo的知识库存储 |
| 部署 | 前端：Vercel / Netlify<br>后端：Railway / Render | 免费额度，支持自动部署 |
| 版本控制 | Git + GitHub | 已有仓库 |

## 3. 功能模块

### 3.1 前台（公开访问）

#### 3.1.1 首页
- **Hero区域**：头像/姓名/职位标签（全栈AI开发工程师）；一句价值主张（例如“8年全栈，2年AI落地，3项发明专利”）
- **核心亮点**：图标卡片展示（全栈+AI能力/专利数量/提效60%创新奖/技术栈）
- **部分项目展示**：精选3个项目，带图片/图标、标题、简短描述 -> 点击跳转作品详情页
- **最新博文**：展示最近2篇博文标题、发布日期、摘要 -> 点击跳转博文详情页
- **AI快速体验入口**：一个浮动按钮或卡片“试试AI助手”，打开对话框（可集成简单的RAG问答）

#### 3.1.2 作品集模块
- **作品列表页**：网格/列表展示所有项目，每个项目展示封面图、标题、技术标签、简介
- **作品详情页**：
  - 项目概述与角色
  - 技术栈与难点
  - 量化成果（提效xx%，覆盖数据量，获奖）
  - 专利信息（专利名称、状态、本人为第一发明人）
  - 可展示项目架构图、截图（若允许）
  - **AI Demo延伸**（可选）：如果该项目包含AI能力，提供一个“体验AI功能”的按钮，调用后端AI接口（如语音质检模拟、推荐话术生成）

#### 3.1.3 博文模块
- **博文列表页**：分页展示文章卡片（标题、发布时间、阅读时长、标签、摘要）
- **博文详情页**：
  - 支持Markdown渲染
  - 显示发布时间、阅读量（可选）
  - 底部“点赞”、“评论”功能（可选，但加分）
  - 评论区（简单实现，需用户昵称+内容，存储到数据库；管理后台审核或直接显示）

#### 3.1.4 AI体验专区（独立页面或侧边栏）
提供不少于2个AI交互Demo（可逐步迭代，MVP至少1个）：
- **Demo1：智能RAG问答（基于个人知识库）**  
  - 用户输入问题 → 后端检索预置的知识库（如个人介绍、项目细节、技术文章）→ 调用大模型生成回答，并展示检索到的来源片段。
  - 目的：展示RAG落地能力。
- **Demo2：虚拟Agent推荐话术生成（可选）**  
  - 用户选择用户角色（决策者/使用者/观望者）→ Agent根据角色生成推荐话术。
  - 展示多智能体决策理念（来自专利2）。
- **其他扩展**：如代码补全、SQL查询助手等。

#### 3.1.5 关于我
- 个人简介（文字+照片）
- 工作经历（时间线简表）
- 技能图谱（词云或进度条或标签云）
- 社交链接（GitHub、领英、邮箱）

### 3.2 管理后台（需登录）
用于动态管理作品、博文、AI知识库（可选）。
- **登录页**：输入密码（或邮箱+密码），后端验证，返回JWT。
- **仪表盘**：统计访问量、评论数等（可简单展示）。
- **作品管理**：增删改查作品（标题、封面图URL、内容、技术标签、排序权重等）。
- **博文管理**：Markdown编辑器（可集成SimpleMDE或TipTap）发布/编辑/删除，支持保存为草稿。
- **知识库管理**（为AI Demo1服务）：添加/编辑/删除文档片段（标题、内容），用于RAG检索。

> 可选：评论审核面板（如实现评论功能）。

## 5. API 设计（RESTful）

基础路径：`/api/v1`

### 5.1 公开接口

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/projects` | 获取所有已发布作品列表（支持分页、标签筛选） |
| GET | `/projects/{slug}` | 获取单个作品详情 |
| GET | `/posts` | 获取已发布博文列表（分页） |
| GET | `/posts/{slug}` | 获取博文详情（同时增加 view_count） |
| GET | `/info` | 获取个人基础信息（简介、社交链接、技能等） |
| POST | `/ai/rag` | RAG问答：`{ "question": "xxx" }` → `{ "answer": "...", "sources": [...] }` |
| POST | `/ai/agent/demo` | Agent模拟（可选）：`{ "user_role": "决策者" }` → `{ "suggestion": "..." }` |
| POST | `/comments` | 发布评论（需验证：昵称+内容） |
| GET | `/comments/{post_id}` | 获取某博文的已审核评论列表 |

### 5.2 管理接口（需JWT认证，Header: `Authorization: Bearer <token>`）

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/auth/login` | 登录，返回token |
| POST | `/auth/logout` | 登出（客户端丢弃token即可，服务端可选黑名单） |
| CRUD | `/admin/projects` | 项目的增删改查 |
| CRUD | `/admin/posts` | 博文的增删改查 |
| CRUD | `/admin/knowledge` | 知识库文档管理 |
| GET | `/admin/comments` | 查看未审核评论 |
| PUT | `/admin/comments/{id}/approve` | 审核通过评论 |

> 建议管理接口使用 `/admin` 前缀，并统一进行权限校验。

## 7. AI集成实现要点

### 7.1 RAG问答（Demo1）
- 知识库来源：后台管理中维护的知识文档（`knowledge_docs`表）。
- 检索方式（简化版，无需向量数据库）：
  - 使用关键词匹配（如TF-IDF或简单字符串包含）找出Top 2相关文档。
  - 或使用 `sentence-transformers` 离线模型生成向量（但会增加部署复杂度），初期推荐轻量级检索：将用户问题转小写，对每个文档内容计算包含的关键词权重。
- 生成应答：构造提示词：
  ```
  你是一个个人助理，基于以下资料回答用户问题。
  资料：
  {source1}
  {source2}
  问题：{question}
  请只根据资料回答，若资料无法回答则说“资料库中暂无相关信息”。
  ```
- 返回内容：`answer` + `sources`（匹配到的文档标题）。

### 7.2 Agent模拟（Demo2，可选）
- 基于专利2（多智能体决策）的简化展示：
  - 用户选择角色类型（决策者/使用者/观望者）
  - 后端根据角色从预设话术库中检索（或调用大模型生成）推荐话术。
- 可增加简单历史记忆（存储最近3轮对话）。

> 注意：所有AI调用需记录日志，并做异常处理（API Key失效、超时等），返回友好提示。

## 8. 开发与部署步骤建议

### 8.1 第一阶段：MVP（1-2周）
- 前端：首页 + 作品列表/详情（数据从后端的Mock API或硬编码过渡）。
- 后端：项目、博文的基础CRUD（含管理接口），数据库使用SQLite。
- 部署前端到Vercel，后端到Railway（本地开发可先用ngrok暴露后端测试）。
- 完成AI RAG Demo（简化检索）。

### 8.2 第二阶段：博文与评论（1周）
- 增加博文模块，支持Markdown展示。
- 评论功能（前端可见，后台审核）。

### 8.3 第三阶段：AI扩展与美化（1周）
- 增加Agent Demo。
- 前端UI完善（动画、响应式细节）。
- 性能优化（图片懒加载、API缓存）。

### 8.4 第四阶段：管理后台完善与安全（可选）
- 添加JWT刷新机制。
- 增加访问统计（如使用`fastapi-limiter`做限流，`user_agents`记录）。
